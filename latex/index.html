Mathematics involves many symbols and layouts that are easy to write by hand but difficult to produce using a keyboard and mouse. For example:

$$ -b \pm \sqrt{b^2 -4ac} \over 2a $$

One technique is to use a wysiwyg editor ("what you see is what you get"), such as the one that comes with Microsoft Word. Wysiwyg editors are good for the occasional piece of simple mathematics, but they are unwieldy for more extensive and complex mathematics.

Many mathematicians and scientists prefer to use a markup language, which allows them to type mathematics using plain text and have it automatically rendered into the appropriate symbols and layouts.

One such language is TeX (pronounced "tek"), developed in the 1970s by computer scientist and mathematician Donald Knuth. An easier-to-use and more popular version is LaTeX (pronounced "lay tek"), developed in the 1980s by computer scientist Leslie Lamport.

Here's how we might use LaTeX to write the expression above:

 
latex
 

This might seem daunting, but once you get the hang of it you'll find it pretty intuitive. Notice that if you read the mathematical expression at the top of this page out loud, and then read the snippet of LaTeX out loud, you say much the same thing. This is no accident - LaTeX is designed to be written in the way we naturally read and speak mathematics.

Using LaTeX has many advantages - it's worth taking some time to learn it.

 
Using LaTeX
 

Many web applications allow you to add LaTeX to your regular text, simply by enclosing it in dollar signs. Browsers will automatically convert your LaTeX into the appropriate symbols and layout.

Depending on how you want your mathematics to be displayed, you can use either single dollar signs or double dollar signs.

 

Using single dollar signs

If you surround your LaTeX with single dollar signs it will be displayed inline. For example:

 
markdown
 

Using double dollar signs

If you surround your LaTeX with double dollar signs it will be displayed as a block, on its own line. For example:

 
markdown
 

Using a mixture

You can use a mixture of the two:

 
markdown
